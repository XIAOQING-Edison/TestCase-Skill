# Skill 决策案例库（正例 / 反例 / 冲突例）

本文件用于约束 `testcase-generator` 在复杂输入下的决策行为，避免“看起来正确但实际越界”的输出。

## 使用方式

- 先判断文档类型与信息完整度（L1 / L2 / L3）
- 再按案例模式决定：继续生成 / 先追问 / 停止生成
- 对于冲突规则，优先输出待确认与反向校验用例

---

## 正例

### 正例 1：结构化需求文档，规则清晰

- 输入片段：按“功能点 + 规则列表 + 提示文案”组织
- 判定：L1
- 动作：直接生成功能测试用例
- 要求：输出常规初稿，可包含规则型用例

### 正例 2：OpenAPI 文档，成功响应与错误响应完整

- 输入片段：包含 `paths`、成功/失败响应、schema
- 判定：L1
- 动作：直接生成 API 用例
- 要求：断言来源可标记为文档定义，假设项应较少

### 正例 3：需求文档包含“支持/不支持”明确描述

- 输入片段：“支持扫码录入”“不支持换行搜索”
- 判定：L1
- 动作：直接生成正向 + 反向用例
- 要求：不可同时输出互斥标题

---

## 反例

### 反例 1：文档未定义错误码却输出精确错误码

- 输入片段：仅有成功流程描述，无错误响应定义
- 禁止行为：输出“40001/Token 失效”等确定性断言
- 正确动作：标记 `待确认`，并标注断言来源为文档缺失

### 反例 2：仅写“支持搜索”却扩写为多种搜索能力

- 输入片段：文档只出现“支持搜索”
- 禁止行为：擅自扩写“模糊搜索/拼音搜索/组合搜索”
- 正确动作：生成通用搜索可用性用例，扩展项标记待确认

### 反例 3：文档冲突时输出单一确定结论

- 输入片段：同文档出现“支持”与“不支持”描述
- 禁止行为：忽略冲突并给出单边结论
- 正确动作：按优先级处理并提示冲突，生成反向校验用例

---

## 冲突例

### 冲突例 1：支持换行搜索 vs 不支持换行搜索

- 判定：规则冲突
- 优先级：明确否定 > 明确肯定
- 处理：输出“不支持换行搜索”主结论，并提示冲突待确认

### 冲突例 2：默认显示 vs 默认隐藏

- 判定：状态冲突
- 优先级：以最新版本说明或明确否定语句优先
- 处理：输出“默认状态待确认”并生成切换行为校验用例

### 冲突例 3：可叠加 vs 不可叠加

- 判定：优惠规则冲突
- 优先级：不可叠加 > 可叠加
- 处理：先按互斥处理并标记待确认，避免输出过度乐观结果

## 复杂 PRD 参考案例（当前用户样本）

### 样本：`【PRD】自助收银加价购v1.0.md`

- 特征：
  - 图文混合，含大量图片占位内容
  - 长段规则与表格规则并存
  - 互斥/叠加规则复杂，且含会议纪要补充项
- 建议判定：L2
- 建议动作：
  - 先追问关键歧义（叠加优先级、默认状态、限购口径）
  - 若用户先看初稿，输出低置信度初稿 + 待确认项清单
